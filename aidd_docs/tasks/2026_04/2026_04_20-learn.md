# Apprentissage et Documentation

## Contexte
Capturer les connaissances acquises lors de l'implémentation du Master Plan.

## Leçons Apprises

### 1. Intégration d'Ollama
- **Prompt Engineering** : L'importance de spécifier un format strict pour les sorties du LLM (ex: `Niveau1/Niveau2/Niveau3`).
- **Validation** : Toujours valider les sorties du LLM pour éviter les erreurs de format ou les caractères interdits.
- **Fallback** : Prévoir un mécanisme de fallback vers des règles par défaut si le LLM échoue.

### 2. Apprentissage Incrémental
- **Gestion des Classes** : `partial_fit` de scikit-learn nécessite que les classes soient cohérentes avec le modèle existant. Solution : reconstruire le modèle si de nouvelles classes sont ajoutées.
- **Logs** : Les logs sont essentiels pour suivre les mises à jour du modèle et diagnostiquer les problèmes.
- **Sauvegarde** : Sauvegarder le modèle après chaque mise à jour pour éviter les pertes de données.

### 3. Tests Unitaires
- **Mocking** : Utiliser `unittest.mock.patch` pour simuler les appels à Ollama et tester les fallback.
- **Edge Cases** : Tester les fichiers corrompus et les entrées invalides pour garantir la robustesse.
- **Logs** : Capturer les logs dans les tests pour vérifier que les messages sont générés correctement.

### 4. Documentation
- **Diagrammes Mermaid** : Utiliser des diagrammes pour expliquer les workflows complexes.
- **Exemples** : Inclure des exemples de code et de configuration pour faciliter l'utilisation.
- **Améliorations Futures** : Documenter les améliorations possibles pour guider les contributions futures.

## Bonnes Pratiques

### Code
- **Modularité** : Séparer la logique en fonctions distinctes (ex: `propose_path`, `record_decision`).
- **Validation** : Valider les entrées et les sorties à chaque étape.
- **Logs** : Utiliser des logs pour suivre l'exécution et diagnostiquer les problèmes.

### Tests
- **Couverture** : Tester les cas principaux et les edge cases.
- **Mocking** : Utiliser des mocks pour isoler les dépendances externes (ex: Ollama).
- **Assertions** : Vérifier les logs et les fichiers générés.

### Documentation
- **Clarté** : Expliquer les concepts complexes avec des exemples.
- **Diagrammes** : Utiliser des diagrammes pour visualiser les workflows.
- **Exemples** : Fournir des exemples de code et de configuration.
